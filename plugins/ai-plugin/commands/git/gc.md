description: 创建git commit提交命令，接收一个参数作为标题，标题参数为空需要进行提示。
---

1. 分析当前git变更内容，不需要对`*.md` 或者`*_test.go`结尾的文件分析，分析主要从数据结构、业务逻辑、流程与接口等方面进行，并按照如下格式生成 comment

### comment格式

```markdown
$ARGUMENTS  \n  #标题

- 整体变更描述  #固定格式，与标题之间保持空一行

- 变更内容简介  #固定格式

1. 
2. 
3.

- 变更文件列表  #固定格式

1.
2.
3.

Generated by jww with AI #固定格式

```markdown

2. git add变更文件，包括`*_test.go`结尾的文件，不包括`*.md`结尾文件
3. 以 "$ARGUMENTS" 为标题创建git提交，执行`git commit -m "$(cat <<'EOF' comment EOF)"`命令

## 注意事项
 - 不要修改代码
 - comment中标题与后面的内容中间要空一行
