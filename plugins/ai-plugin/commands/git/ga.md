description: 创建git commit amend提交命令，可选携带一个标题参数。
---

1. 分析最近一次commit提交和当前git变更内容，不需要对`*.md` 或者`*_test.go`结尾的文件分析，分析主要从数据结构、业务逻辑、流程与接口等方面进行，并按照如下格式生成comment

### comment格式

```markdown
$ARGUMENTS  \n   #标题

- 整体变更描述  #固定格式，与标题之间保持空一行

- 变更内容简介  #固定格式

1. 
2. 
3.

- 变更文件列表  #固定格式

1.
2.
3.

Generated by jww with AI #固定格式

```markdown

2. git add变更文件，包括`*_test.go`结尾的文件，不包括`*.md`结尾文件
3. 如果 "$ARGUMENTS" 不为空，则需要修改原commit comment标题，并更新comment其它内容；如果为空，标题不变，仅更新comment内容
4. 不要修改原commit中change-id，保留相同change-id
5. 执行`git commit --amend -m "$(cat <<'EOF' comment EOF)"`命令，并将comment内容添加进去

## 注意事项
 - 不要修改代码
 - 不要修改commit中的change-id
 - comment中标题与后面的内容中间要空一行
