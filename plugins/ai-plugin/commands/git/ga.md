---
description: 创建git commit amend提交命令 
---

1. 分析最近一次commit提交和当前git变更内容，不需要对`*.md` 或者`*_test.go`结尾的文件分析，分析主要从数据结构、业务逻辑、流程与接口等方面进行，并按照如下格式生成comment

```markdown

- 整体变更描述  #固定格式

- 变更内容简介  #固定格式
1. 
2. 
3.

- 变更文件列表  #固定格式
1.
2.
3.

Generated by jww with AI #固定格式

```markdown

2. git add变更文件，包括`*_test.go`结尾的文件，不包括`*.md`结尾文件
3. 如果 "$ARGUMENTS" 不为空，则需要修改原commit标题，并更新comment内容；如果为空，标题不变，仅更新comment内容
4. 不要修改原commit中change-id，保留相同change-id

## 注意事项
 - 不要修改代码
